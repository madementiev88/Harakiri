# HARAKIRI — Telegram-бот для барбершопа

## О проекте

**HARAKIRI** — Telegram-бот для самостоятельной онлайн-записи клиентов в барбершоп Harakiri.

**Проблема:** администратор тратит время на обработку звонков и ручную запись. Клиенты не могут записаться в нерабочее время. Мастера не имеют удобного инструмента для просмотра своего расписания.

**Решение:** бот закрывает весь цикл записи — от выбора мастера до подтверждения визита — без участия администратора.

**Аудитория:**
- Клиенты барбершопа (основная аудитория)
- Мастера (просмотр расписания, управление слотами)
- Администратор (контроль всех записей, управление)

## Функционал

### Для клиента

- **Онбординг** — приветственное сообщение при первом запуске; бот предлагает поделиться номером телефона (опционально, запись работает и без него)
- **Выбор мастера** — профиль с фото, именем, должностью "Барбер" и средним рейтингом; на каждом шаге доступна кнопка "Выбрать другого мастера"
- **Избранный мастер** — назначается автоматически после первого завершённого визита; при следующей записи бот предлагает его первым
- **Каталог услуг** — стрижка, борода, комбо, детская стрижка и другие услуги с ценами
- **Запись на приём** — выбор мастера → услуга → дата и время → подтверждение
- **Напоминания** — автоматически за 24 часа и за 1 час до визита
- **Отмена записи** — клиент может отменить запись не позднее чем за 1 час до начала
- **Оценка и быстрая перезапись** — через 10 минут после окончания слота бот отправляет одно сообщение: запрос оценки (1–5 звёзд + комментарий) и кнопку быстрой перезаписи к тому же мастеру на ту же услугу; если клиент отмечен как no-show — сообщение не отправляется
- **Информация** — адрес, часы работы, контакты

### Для мастера

- **Уведомление о новой записи** — приходит в личку сразу после каждого нового бронирования
- **Утренний дайджест** — расписание на день отправляется каждое утро в **09:00**
- **/расписание** — список клиентов на любой день в формате: имя, телефон, время, услуга
- **Блокировка слотов** — выходной, отпуск, болезнь; при блокировке занятых слотов клиенты получают уведомление автоматически
- **Отметка no-show** — фиксация клиентов, не пришедших на запись

### Для администратора

- **Уведомление о новой записи** — приходит в личку сразу после каждого нового бронирования
- **Уведомление о текстовом отзыве** — когда клиент оставляет комментарий к оценке, admin получает его в личку (мастер и другие клиенты не видят)
- **Расписание всех мастеров** — команда `/allschedules`: выбор даты → сводка по всем мастерам на день в формате: имя мастера → список записей (время — имя клиента — услуга)
- Добавление мастера командой `/addmaster` с указанием Telegram ID или @username
- Удаление мастера командой `/removemaster` (см. сценарий увольнения)
- Управление профилями мастеров — фото, имя, описание (только администратор)
- Настройка рабочих часов и длительности слота
- Блокировка слотов мастера с автоматическим уведомлением клиентов
- Ручное создание и отмена записей
- Просмотр статистики: загрузка по дням, топ мастеров

### Маркетинг и удержание

- **Push-уведомления** — рассылка клиентам, которые не приходили более 1 месяца; отправляется ежедневно в **15:00 по МСК**

## Бизнес-правила

### Расписание и слоты

- Рабочие часы: **10:00 — 21:00**
- Длительность слота: **1 час** (настраивается администратором)
- Подход: **шаблон недели + исключения**

**Как работает:**
1. Admin один раз задаёт стандартную рабочую неделю для каждого мастера (дни, часы начала и конца, выходные)
2. Слоты автоматически генерируются на **30 дней вперёд** на основе шаблона
3. При необходимости admin переопределяет конкретный день или блокирует период

**UX для администратора:**
```
Управление расписанием → Выбрать мастера
  ├── Шаблон недели       → настроить стандартные часы по дням
  ├── Изменить день       → выбрать дату из календаря, задать новые часы
  └── Заблокировать период → выбрать диапазон дат (отпуск, больничный)
```

### Транзакционная блокировка слота

Предотвращает двойное бронирование при одновременных запросах:

1. Клиент выбирает слот → слот получает статус **"reserved"** на **10 минут**
2. В течение 10 минут слот не виден другим клиентам
3. Клиент подтверждает запись → статус меняется на **"booked"**
4. Если клиент не завершил бронирование за 10 минут → слот освобождается автоматически
5. На уровне БД используется `SELECT FOR UPDATE` — атомарный захват слота исключает race condition

### Правила отмены

- Клиент может отменить запись **не позднее чем за 1 час** до начала
- После дедлайна отмена возможна только через администратора

### Избранный мастер

- Назначается **автоматически** после первого завершённого визита клиента
- При следующей записи бот предлагает этого мастера первым
- Если у избранного мастера нет свободных слотов — бот показывает ближайшие доступные слоты у него и добавляет кнопку **"Выбрать другого мастера"**

### Оценка мастера

- Через **10 минут после окончания слота** бот отправляет одно сообщение с запросом оценки и кнопкой быстрой перезаписи
- Если клиент отмечен мастером как **no-show** — сообщение не отправляется
- Звёздная оценка (1–5) отображается в профиле мастера как средний рейтинг
- Текстовый комментарий виден **только администратору** через уведомление в личку

### Сценарий увольнения мастера

1. Admin выполняет `/removemaster`
2. Бот автоматически находит всех клиентов с будущими записями к этому мастеру
3. Все будущие записи отменяются
4. Каждому клиенту отправляется уведомление об отмене с предложением записаться к другому мастеру
5. У клиентов, у которых этот мастер был в избранных, выбор сбрасывается

### Сценарий болезни мастера

1. Администратор блокирует слоты мастера
2. Бот автоматически находит клиентов с активными записями к этому мастеру
3. Каждому клиенту отправляется уведомление об отмене с вариантами:
   - Выбрать свободный слот у другого мастера на этот же день
   - Выбрать ближайший свободный слот у любого мастера
   - Встать в очередь к тому же мастеру
4. Записи автоматически отменяются
5. Когда admin разблокирует слоты мастера — бот автоматически уведомляет всех из очереди

### Обработка отписок

- Если клиент заблокировал бота, Telegram возвращает ошибку `TelegramForbiddenError`
- Бот устанавливает флаг `is_blocked = True` в записи клиента в БД
- Клиенты с флагом исключаются из всех рассылок и уведомлений
- Если клиент снова пишет боту — флаг сбрасывается в `is_blocked = False`

### Персональные данные клиента

Бот предлагает поделиться номером телефона при онбординге. Телефон опционален — запись работает без него.
Если телефон предоставлен, он виден администратору и мастеру в расписании.
Имя, дата рождения и другие личные данные не запрашиваются и не хранятся.

## Регистрация персонала

### Администратор
Telegram ID администратора задаётся в `.env` при деплое. Смена администратора — через обновление конфига и перезапуск.

### Мастер
1. Мастер получает уведомление в личку о том, что добавлен в систему
2. С этого момента мастеру доступны команды `/расписание`, блокировка слотов, отметка no-show

## Технический стек

| Компонент | Технология |
|---|---|
| Язык | Python 3.11 |
| Telegram-фреймворк | aiogram 3.x (FSM для диалогов) |
| FSM-хранилище | Redis (сохранение состояний диалогов между перезапусками) |
| База данных | PostgreSQL |
| ORM и миграции | SQLAlchemy + Alembic |
| Конфигурация | pydantic-settings (управление через .env) |
| Планировщик задач | APScheduler с SQLAlchemy job store (задачи хранятся в PostgreSQL, не теряются при перезапуске контейнера) |
| Reverse proxy | Nginx (webhook-режим) |
| Мониторинг ошибок | Sentry |
| Деплой | Docker + docker-compose |
| Резервное копирование | pg_dump по расписанию (ежедневно в 03:00), хранение последних 7 дней в Яндекс Object Storage (S3-совместимый) |
